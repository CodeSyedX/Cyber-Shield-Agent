[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's walk through the architecture of our agent.
Every query enters through the START node and is immediately scanned by the security_checkpoint.
The checkpoint filters prompt injection and redacts personal data, routing violations to a separate security_event_handler.
Safe queries continue to our orchestrator agent.
The orchestrator delegates work: it sends source code to the secret_scanner and package lists to the vuln_analyzer.
Both sub-agents query our local MCP server tools—regex_scanner and check_cve_database—for fast matching.
The orchestrator combines findings and passes them to the review_gate.
If high-severity vulnerabilities are found, the review_gate issues a RequestInput interrupt, asking for human confirmation.
Once approved, the generate_report node compiles the final Markdown document.
